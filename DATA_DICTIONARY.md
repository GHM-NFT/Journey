THese Are the collection tabs, they all have the same columns, scripts and desired inputs/outputs. The columns are labelled under Manual/Auto/ Auto if empty, these refer to the scripts in operation filling in cells.

Column name + meaning + type (string, int, bool, date, json) + example value (GHM Manifest categories/columns (Manual, Auto)).

MANUAL (per-token entries you provide)
* token_id — on-chain ID or planned number. (Unique per token — not copy/paste.)
* Title/Name — artwork/character title. (Usually unique — not copy/paste.)
* Pantheon — collection/family (e.g., “Sea Deities”). (Often constant per sheet → copy/paste whole column.)
* Frame — frame style text. (Often constant within a series → copy/paste.)
* Pallette — colorway name. (Usually constant per variant or run → copy/paste in blocks.)
* Format/Medium — e.g., “Round canvas + video”. (Often constant per series → copy/paste.)
* Stylisation (optional) — style notes. (Often constant per series → copy/paste.)
* Variant (optional) — A/B/C, if you keep it. (Varies — sometimes copy/paste in blocks.)
* License_URL — link to license/terms. (Constant per brand/collection → copy/paste whole column.)
* image_filename — poster/still filename. (Unique per token — not copy/paste. Use the guesser to speed up.)
* animation_filename (optional) — video/3D filename. (Unique per token — not copy/paste.)
* background_color (optional) — 6-digit hex (no #). (Often constant per collection → copy/paste.)
* title_en, description_en — English text. (Usually varies; can copy/paste if same blurb for a run.)
* title_zh-Hans, description_zh-Hans (optional) — Chinese text. (Often shared per collection → copy/paste.)
* edition_size — 1 for 1/1, N for 1155. (Constant per drop type → copy/paste.)
* token_range (optional) — planning note like 1001–1050. (Per run → copy/paste.)

AUTO (fully generated by scripts)
* name_final — uses token_name if present, else “Title/Name — Edition_Type #token_id”.
* slug — URL-friendly from name_final.
* alt_text_en, alt_text_zh-Hans — concise alt text built from traits.
* attributes_json — canonical traits + optional extras (Stylisation, Medium, License).
* image_mime, image_bytes — from exif scan/merge.
* animation_mime, animation_bytes — from exif scan/merge.
(No manual typing or copy/paste for any of the above.)

AUTO-IF-EMPTY (canonical traits mapped from your human columns)
* Series ← from Pantheon if blank.
* Character ← from Title/Name if blank.
* Character_Variant ← from Variant if blank.
* Frame_Style ← from Frame if blank.
* Colorway ← from Pallette if blank.
* Edition_Type ← from Tier (if used) if blank.
* Medium ← from Format/Medium if blank.
(Once your human/source fields are filled, these back-fill automatically—no copy/paste needed.)
CONTROL SHEET — MANUAL ONCE (project/collection-level settings)
(Set once per collection or series; most are copy/paste whole column for that collection.)
* standard — 721 or 1155. (copy/paste)
* chain — ETH, Base, Polygon. (copy/paste)
* royalty_bps — e.g., 750. (copy/paste)
* royalty_recipient — payout address. (copy/paste)
* operator_filter — off (recommended). (copy/paste)
* operator_policy_note — short royalty/listing note. (copy/paste)
* freeze_policy — e.g., “Freeze 72h post-mint”. (copy/paste)
* freeze_date — planned freeze date. (copy/paste)
* qa_owner — who signs off. (copy/paste)
* contract_factory — Manifold/Zora/Custom. (copy/paste)
* collection_address — contract address. (copy/paste)
* collection_slug — marketplace slug. (copy/paste)
* allowlist_source, claim_start, claim_end, per_wallet_cap, reserve_supply (optional) — only if you run allowlists/claims. (copy/paste when used)

GLOBALS — MANUAL ONCE (set & forget)
* default_locale — en-GB. (copy/paste not needed)
* secondary_locale — e.g., zh-Hans, el, ja. (copy/paste per collection)
* onchain_language — en. (set once)
* site_localization — true. (set once)

2. Frame, Pallette, Format/Medium (and Tier if you use it)
3. image_filename (and animation_filename if applicable)
4. License_URL, edition_size (and background_color if you want it visible)
